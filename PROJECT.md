# OrbitaMarket - Планирование проекта

## Цель проекта

Создание платформы для торговли спутниковыми данными с асинхронным взаимодействием.

**Ключевые задачи:**
- Реализовать Payments Service для управления счетами пользователей
- Реализовать Orders Service для обработки заказов на спутниковые снимки
- Обеспечить надёжное асинхронное взаимодействие через Kafka
- Создать API Gateway для единой точки входа
- Применить паттерны Inbox и Outbox

### Этап 4: Apache Kafka Integration (Неделя 3)

**Даты:** 2026-07-04 — 2026-07-06

**Задачи:**

- Настройка Kafka в Docker Compose
- Создание топиков:
    - `order-payment-requested` (3 партиции)
    - `order-payment-completed` (3 партиции)
    - `order-payment-failed` (3 партиции)
- Producer в Orders Service (отправка OrderPaymentRequested)
- Consumer в Payments Service (обработка OrderPaymentRequested)
- Producer в Payments Service (отправка OrderPaymentCompleted/Failed)
- Consumer в Orders Service (обработка результатов оплаты)
- Kafka UI для мониторинга

**Результат:** Асинхронное взаимодействие между сервисами через Kafka

**Критерии готовности:**

- ✅ При создании заказа событие уходит в Kafka
- ✅ Payments Service получает и обрабатывает событие
- ✅ Результат оплаты возвращается в Orders Service
- ✅ Статус заказа обновляется на PAID/PAYMENT_FAILED
- ✅ Топики видны в Kafka UI

### Этап 8: Сервис автоматической выдачи заказов (Неделя 6)

**Даты:** 2026-07-20 — 2026-07-27

**Задачи:**

- Создание нового микросервиса `delivery-service`
- Настройка PostgreSQL (delivery_db)
- Интеграция с MinIO/S3 для хранения спутниковых снимков
- Consumer для топика `order-payment-completed`
- Генерация pre-signed URLs с TTL (24 часа)
- REST API endpoints:
- `GET /api/v1/delivery/{orderId}` — получить ссылку на скачивание
- `GET /api/v1/delivery/{orderId}/status` — статус доставки
- Интеграция с Redis для кэширования
- Email-уведомления о готовности данных
- Dockerfile и интеграция в docker-compose
- Swagger документация


**Результат:** Автоматическая выдача заказанных данных после успешной оплаты
  

**Критерии готовности:**

- ⏳ При статусе заказа PAID автоматически генерируется ссылка на скачивание
- ⏳ Ссылка имеет TTL 24 часа и защищена подписью
- ⏳ Клиент получает email-уведомление о готовности данных
- ⏳ API возвращает статус: PROCESSING / READY / FAILED
- ⏳ Снимки хранятся в MinIO, метаданные — в PostgreSQL
